Q-Learning

Q-Learning
📝 בקצרה
Q-Learning הוא אלגוריתם למידת חיזוקים טבלאי וחסר מודל הלומד פונקציית ערך Q(s,a) המעריכה את התגמול העתידי הצפוי מביצוע פעולה a במצב s. האלגוריתם מעדכן את הטבלה לפי משוואת בלמן עם חוקיות off-policy: הוא לומד את המדיניות האופטימלית גם כאשר הסוכן חוקר באמצעות אסטרטגיה אחרת כמו ε-greedy. עם מספיק ביקורים בכל זוג (מצב, פעולה) ו-α דועך, Q מתכנס לערך האופטימלי Q*. מתאים למרחבי מצבים ופעולות בדידים וקטנים.
📐 נוסחה
Q(s,a) ← Q(s,a) + α·[r + γ·max_a' Q(s',a') − Q(s,a)]
📊 המחשה ויזואלית
💻 קוד פייתון
import numpy as np, gymnasium as gym
env = gym.make("FrozenLake-v1", is_slippery=False)
Q = np.zeros((env.observation_space.n, env.action_space.n))
alpha, gamma, eps = 0.1, 0.99, 0.1
for _ in range(5000):
    s, _ = env.reset(); done = False
    while not done:
        a = env.action_space.sample() if np.random.rand() < eps else np.argmax(Q[s])
        s2, r, done, trunc, _ = env.step(a)
        Q[s, a] += alpha * (r + gamma * np.max(Q[s2]) - Q[s, a])
        s = s2
policy = np.argmax(Q, axis=1)  # מדיניות חמדנית נלמדת

  • Off-policy: לומד מדיניות אופטימלית תוך חקירה עם מדיניות אחרת
  • פשוט להבנה ולמימוש, ללא צורך במודל של הסביבה
  • שקוף ולומדים יכולים לבדוק ישירות את ערכי Q בטבלה
  • בסיס תיאורטי לאלגוריתמים מתקדמים כמו DQN ו-Double DQN
✗ חסרונות

  • Maximization bias: מטיה כלפי הערכת יתר של פעולות
  • התכנסות איטית כשהמרחב גדול או התגמולים דלילים
  • רגיש להיפר-פרמטרים α, γ ו-ε

  • הוראת יסודות RL וניסויים אקדמיים
  • פתרון MDP קטן ללא מודל מעבר ידוע
  • בסיס לבניית פתרונות RL מתקדמים יותר

Deep Q-Network

DQN
📝 בקצרה
Deep Q-Network הוא אלגוריתם למידת חיזוק שבו רשת נוירונים מקורבת את פונקציית הערך Q(s,a) במקום טבלה. הרשת מקבלת מצב כקלט ומחזירה ערך Q לכל פעולה אפשרית, מה שמאפשר טיפול במרחבי מצבים עצומים כמו פיקסלים של מסך. DeepMind הציגה אותו ב-2013 והפעילה אותו על משחקי Atari ברמת ביצועים על-אנושית. שני רכיבים קריטיים ליציבות: Experience Replay ו-Target Network.
📐 נוסחה
L(θ) = E[(r + γ·max_a' Q(s',a';θ⁻) − Q(s,a;θ))²]
📊 המחשה ויזואלית
💻 קוד פייתון
import gymnasium as gym
from stable_baselines3 import DQN

env = gym.make("CartPole-v1")
model = DQN("MlpPolicy", env, learning_rate=1e-3, buffer_size=10000, verbose=0)
model.learn(total_timesteps=20000)
obs, _ = env.reset()
action, _ = model.predict(obs, deterministic=True)

  • Target Network מונע אוסילציות בלמידה
  • לומד ישירות מפיקסלים ללא הנדסת פיצ'רים ידנית

Policy Gradient

Policy Gradient
📝 בקצרה
Policy Gradient הוא משפחה של אלגוריתמי למידת חיזוק שמאופטמת ישירות את הפוליסה הסטוכסטית π(a|s;θ) במקום ללמוד פונקציית ערך. האלגוריתם הקלאסי REINFORCE מחשב את הגרדיאנט של התגמול הצפוי לפי lemma הלוג-נגזרת ומעדכן את משקלי רשת הפוליסה בכיוון פעולות שהובילו לתגמול גבוה. השיטה מתאימה למרחבי פעולה רציפים ולפוליסות סטוכסטיות, אך סובלת משונות גבוהה ומדגימות לא יעילה. גרסאות מתקדמות (Actor-Critic, PPO, TRPO) מפחיתות שונות באמצעות baseline או פונקציית ערך עזר.
📐 נוסחה
∇θ J(θ) = E[∇θ log π(a|s;θ) · Gₜ]
📊 המחשה ויזואלית
💻 קוד פייתון
import gymnasium as gym
from stable_baselines3 import PPO  # PPO = Policy Gradient מתקדם

env = gym.make("CartPole-v1")
model = PPO("MlpPolicy", env, verbose=0)
model.learn(total_timesteps=20_000)
obs, _ = env.reset()
action, _ = model.predict(obs, deterministic=False)

A3C

A3C
📝 בקצרה
A3C הוא אלגוריתם Actor-Critic מקבילי שבו מספר עובדים (workers) רצים בו-זמנית בסביבות עצמאיות ומעדכנים אסינכרונית רשת גלובלית משותפת. כל עובד מחשב גרדיאנטים מקומיים על בסיס ניסיונו, שולח אותם לרשת המרכזית ומסנכרן בחזרה את המשקולות. הגיוון בין העובדים מפרק את המתאם בין דגימות עוקבות ומחליף את ה-Replay Buffer של DQN. הגישה הזו האיצה דרמטית אימון RL על CPU לפני שהפכה ל-A2C הסינכרוני.
📐 נוסחה
∇θ' log π(aₜ|sₜ;θ')·A(sₜ,aₜ) + β·∇θ' H(π(·|sₜ;θ'))
📊 המחשה ויזואלית
💻 קוד פייתון
import torch, torch.multiprocessing as mp
from a3c import ActorCritic, Worker  # implementation reference

global_net = ActorCritic(state_dim=4, action_dim=2)
global_net.share_memory()
workers = [Worker(global_net, env_name="CartPole-v1", wid=i) for i in range(mp.cpu_count())]
[w.start() for w in workers]; [w.join() for w in workers]
action = global_net.act(torch.tensor(env.reset()))

📝 בקצרה
DDPG הוא אלגוריתם Actor-Critic מבוסס off-policy ללמידת מדיניות דטרמיניסטית במרחבי פעולה רציפים. הוא משלב רעיונות מ-DQN (Replay Buffer ורשתות יעד) עם Policy Gradient דטרמיניסטי. ה-Actor מפיק פעולה רציפה ישירות, וה-Critic מעריך את ערך הזוג (מצב, פעולה) באמצעות Q-function. רעש חקירה (Ornstein-Uhlenbeck או גאוסי) נוסף לפעולה בזמן האימון.

📝 בקצרה

ResNet-Residual Networks
📝 בקצרה
פריצת הדרך של 2015 שאפשרה לאמן רשתות עמוקות (50, 101, 152 שכבות) ע"י skip connections - קיצורי דרך שמאפשרים לגרדיאנט לזרום בקלות. עדיין מודל סיווג התמונות הנפוץ ביותר ב-2026 לרוב המשימות הקלאסיות.
📄 ניתוח הפייפר - "Deep Residual Learning for Image Recognition" (2015)
המחברים: Kaiming He, Xiangyu Zhang, Shaoqing Ren, Jian Sun (Microsoft Research Asia). arxiv: 1512.03385 · ICLR/CVPR 2016 - Best Paper Award.

הבעיה שהם זיהו — Degradation problem:
  • לפני 2015, מקובל היה: יותר עומק = יותר דיוק
  • הם הריצו ניסוי — CIFAR-10 עם 20 שכבות → 56 שכבות
  • תוצאה מפתיעה: הרשת העמוקה יותר גרועה יותר באימון (!)
  • זה לא overfitting - training error עצמו גבוה יותר
ההשערה שלהם: קשה לרשת ללמוד את הפונקציה הזהות. אם השכבות העמוקות היו רק "מעבירות את הקלט קדימה" — ההעמקה לא הייתה מזיקה. אבל הן לא יודעות לעשות את זה.

הפתרון - Residual Learning:
y = F(x, {W_i}) + x
  • במקום ללמוד את H(x) ישירות — לומדים את הבדל: F(x) = H(x) − x
  • אם ה-optimum הוא זהות — F(x) = 0, קל ללמוד
  • הוספה של skip connection ידני — הזהות מובטחת "בחינם"
התוצאות ב-ImageNet 2015:
  • ResNet-152: 3.57% שגיאת top-5 — עוברת ביצועי אדם
  • ניצחה ILSVRC 2015 עם פער של 1.5% מהמקום השני
  • לראשונה הצליחו לאמן CNN עם 152 שכבות
  • ניסוי מפורסם - ResNet עם 1,202 שכבות (!) — עדיין מתאמן
הבנייה הפנימית - שני סוגי blocks:
  • Basic block (ResNet-18/34): 2× (Conv 3×3 + BN + ReLU) + skip
  • Bottleneck block (ResNet-50/101/152): 1×1 → 3×3 → 1×1 + skip

YOLO

YOLO-You Only Look Once
📝 בקצרה
המשפחה המובילה לזיהוי אובייקטים בזמן אמת. שונה מ-R-CNN בכך שמבצעת זיהוי בצעד אחד (one-stage detector) - מהיר במיוחד. גרסאות חדשות (v8, v9, v10, v11) מ-Ultralytics ו-WongKinYiu שולטות בתחום ב-2024-2026.
🧬 משפחת YOLO - כל הגרסאות
YOLOv1 (2016) — Joseph Redmon:
  • הרעיון המהפכני: גריד יחיד + one-shot detection
  • 7×7 גריד, 2 bounding boxes לתא, 45 FPS על Titan X
  • שגיאה של localization ב-objects קטנים
YOLOv2 / YOLO9000 (2017):
  • Anchor boxes מ-Faster R-CNN
  • BatchNorm על כל שכבה
  • 9,000 קלסות דרך WordTree על ImageNet + COCO
YOLOv3 (2018):
  • Darknet-53 backbone (mixed 3×3 + 1×1 עם residual)
  • 3 סקאלות של detection - מזהה קטן, בינוני, גדול
  • ה-YOLO הכי אמין בהיסטוריה - השימוש הפופולרי ביותר עד היום
YOLOv4 (2020) — Alexey Bochkovskiy:
  • אחרי ש-Redmon פרש בגלל שיקולים אתיים - Alexey המשיך
  • CSPDarknet-53 backbone + PANet neck
  • Mosaic augmentation, CIoU loss, DropBlock
YOLOv5 (2020) — Ultralytics:
  • מיישם של Glenn Jocher ב-PyTorch (לא paper רשמי)
  • הכי פופולרי בקהילה - APIs נוחים, ONNX/CoreML export
  • 5 גדלים: n / s / m / l / x
YOLOv7 (2022):
  • E-ELAN block, model re-parameterization
  • SOTA באיזון FLOPS/dוrion accuracy
YOLOv8 (2023) — Ultralytics:
  • Anchor-free detection (בלי anchor boxes)
  • תמיכה מובנית ב-classification + segmentation + pose
  • CLI ו-Python API אלגנטיים
YOLOv9 / v10 / v11 (2024):
  • YOLOv9: Programmable Gradient Information + GELAN
  • YOLOv10: NMS-free ("real-time end-to-end")
  • YOLOv11: Ultralytics המרוכזת של הרעיונות
YOLO-World (2024):
  • Open-vocabulary - מזהה כל עצם שאתה מתאר בטקסט
  • אין צורך לאמן על קלאסים חדשים
  • ריצה בזמן אמת (בניגוד ל-OWL-ViT האיטי)
YOLO-NAS (2023) — Deci.ai:
  • Neural Architecture Search — הארכיטקטורה מחושבת אוטומטית
  • Quantization-aware מהאתחול (INT8 בלי אובדן דיוק)

CLIP

CLIP-Contrastive Language-Image Pre-training
📝 בקצרה

📐 נוסחה
ℒ = -½ Σᵢ [log(exp(sim(Iᵢ,Tᵢ)/τ) / Σⱼ exp(sim(Iᵢ,Tⱼ)/τ))]

DeepSeek (V3, R1)
📝 בקצרה
סטארטאפ סיני שהדהים את העולם בסוף 2024 / תחילת 2025. DeepSeek V3 - מודל גדול ויעיל; DeepSeek R1 - מודל reasoning שמשתווה ל-o1 של OpenAI במחיר זעיר. המאיים הגדול על דומיננטיות OpenAI בזכות פתיחות והוזלת מחיר ה-API. נכון ל-2026: ה-LLM הזול ביותר.
🏗️ ארכיטקטורה
  • סוג ארכיטקטורה: Decoder-only Transformer מבוסס Mixture-of-Experts (MoE) עם Multi-head Latent Attention (MLA) לדחיסת KV-cache.

  • חלון הקשר (Context Window): עד 128K tokens, עם אימון מורחב באמצעות YaRN לטיפול ברצפים ארוכים.
  • מודאליות: טקסט בלבד (text-only), כולל קוד ומתמטיקה; אין תמיכה רשמית בתמונות, אודיו או וידאו בגרסאות V3 ו-R1 הראשיות.
  • יכולות ייחודיות: R1 הוא מודל reasoning שאומן ב-Reinforcement Learning טהור (GRPO) ללא SFT מקדים, עם chain-of-thought ארוך וגלוי; V3 משתמש ב-Multi-Token Prediction (MTP) ובאימון FP8 לחיסכון משמעותי בעלות.
  • זמינות ורישוי: משקלים פתוחים (open-weights) תחת רישיון MIT, כולל גרסאות distilled קטנות יותר (1.5B עד 70B) המבוססות על Qwen ו-Llama.

Gemma-Google
📝 בקצרה
סדרת מודלים קטנים-בינוניים פתוחים של Google, מבוססים על המחקר ש-Gemini בנוי עליו. Gemma 2 (2024) ו-Gemma 3 (2025) מתחרים בעיקר עם Llama ו-Mistral. גודלים קטנים יחסית (2B, 9B, 27B) - מתאימים ל-self-hosting במחיר נמוך.

Dolphin

Dolphin (Cognitive Computations)
📝 בקצרה
סדרת fine-tunes של Eric Hartford שמסירים את "alignment" המקורי של המודל (uncensored). מבוססים על Llama, Mistral, Mixtral, Qwen, ו-Phi. הרעיון: מודל שלא מסרב לבקשות לגיטימיות בתחומי אבטחה, מחקר, וכתיבה יצירתית. שימושים נפוצים במחקר אקדמי ובפיתוח מערכות אבטחה.
🏗️ ארכיטקטורה
  • סוג ארכיטקטורה: משפחת מודלים מבוססי Decoder-only Transformer, המהווים fine-tune של מודלי בסיס פתוחים כגון Llama 3 / 3.1, Mistral, Mixtral (MoE) ו-Qwen 2.5.
  • גודל פרמטרים: מגוון גדלים בהתאם למודל הבסיס - החל מ-2.9B (Dolphin 3.0 Llama 3.2) ועד 70B (Dolphin 2.9 Llama 3 70B), כולל וריאנטים של Mixtral 8x7B ו-8x22B עם ארכיטקטורת Mixture-of-Experts.
  • חלון הקשר (Context Window): נע בין 8K ל-128K טוקנים, תלוי במודל הבסיס - גרסאות מבוססות Llama 3.1 ו-Qwen 2.5 תומכות ב-128K tokens.
  • מודאליות: Text-only (טקסט בלבד), ללא תמיכה מובנית בראייה או באודיו; קלט ופלט טקסטואלי בלבד.
  • שיטת אימון: Supervised Fine-Tuning (SFT) על דאטהסט Dolphin הקנייני, המבוסס על נתוני Orca, Samantha ו-Dolphin-Coder, עם הסרת alignment ושימוש ב-DPO/RLHF בגרסאות מתקדמות.
  • יכולות ייחודיות: מודלים uncensored ו-steerable עם מערכת system prompts גמישה, התמחות ב-instruction following, agentic workflows, function calling ו-coding, ומיועדים להפעלה מקומית תחת רישיון Apache 2.0 / MIT.

OWL-ViT - זיהוי אובייקטים בטקסט חופשי

OWL-ViT · Open-Vocabulary Detection
📝 בקצרה
OWL-ViT ("Open-World Localization with ViT") של Google Research (2022) - הצעד הראשון של open-vocabulary object detection.

הבעיה הקלאסית:
  • YOLO / Faster R-CNN מזהים קלסות סגורות (80 קטגוריות COCO)
  • רוצה לזהות "פינגווין עם קסדה"? צריך לאמן מחדש על דאטהסט חדש
הפתרון של OWL:
  • מבוסס CLIP (Vision Transformer + Text encoder) של OpenAI
  • הוסיפו detection head ל-ViT (bounding boxes + class scores)
  • הקלט: תמונה + טקסט חופשי (בכל שפה!)
  • הפלט: boxes + scores של matching בין ה-embeddings
איך זה עובד:
  1. ViT מקודד את התמונה לתוך גריד של patches
  2. כל patch מקבל embedding וגם bounding box prediction
  3. Text encoder מקודד את ה-query
  4. Cosine similarity בין text embedding ל-patch embeddings
  5. Threshold + NMS → boxes סופיים
🎯 אנלוגיה - חיפוש במאגר תמונות של גוגל
תדמיין חיפוש בגוגל תמונות - אתה מקליד "כלב עם כובע ג’ינס" ומקבל אלפי תמונות רלוונטיות. גוגל לא אומנה מיוחד על "כלב עם כובע ג’ינס" - היא מבינה טקסט ומקשרת ל-visual features.

OWL-ViT זו אותה יכולת, רק בתוך תמונה בודדת:
  • מקבל תמונה + טקסט
  • מחפש בתוך התמונה איפה יש התאמה
  • מסמן bounding box על ה-match
לפני OWL: "אני צריך לזהות כלב עם כובע ג’ינס" → אמן דאטהסט מיוחד. אחרי OWL: "אני צריך לזהות כלב עם כובע ג’ינס" → הקלד את זה, קבל תוצאה.

import numpy as np

tracks = []
frame_idx = 0
while cap.isOpened():
    ret, frame = cap.read()
    if not ret: break
    results = model.track(frame, persist=True, verbose=False)
    for det in results[0].boxes:
        if det.id is None: continue
        tracks.append({
            "id": int(det.id[0]),
            "class": model.names[int(det.cls[0])],
            "frame": frame_idx,
            "box": det.xyxy[0].tolist(),
        })
    frame_idx += 1

def search(query_class, query_color=None):
    return [t for t in tracks if t["class"] == query_class]

people = search("person")
