מרחב מדגם

Sample Space (Ω)
Ω = { כל התוצאות האפשריות }
💡 בקצרה
אוסף כל התוצאות האפשריות של ניסוי או תופעה. נקודת המוצא לכל חישוב הסתברותי. בדאטה: מרחב המדגם הוא לרוב כל השורות בטבלה (כל הדגימות האפשריות).
📊 המחשה ויזואלית
🎯 אנלוגיה 1
כל המנות בתפריט מסעדה
מסעדה עם תפריט של 50 מנות. אם מישהו יזמין מנה בלי שתדע מה - מרחב המדגם של ההזמנה הוא 50 המנות. כל מנה היא 'תוצאה אפשרית'. מרחב המדגם הוא רשימת כל האפשרויות.
📐 מיפוי לנוסחה
  • התפריט = Ω (מרחב המדגם)
  • כל מנה בתפריט = תוצאה אפשרית
  • |Ω| = 50 (מספר התוצאות האפשריות)
  • כל הזמנה היא 'דגימה' אחת מ-Ω
💻 כך זה נראה בקוד
# הטלת קובייה
Omega = {1, 2, 3, 4, 5, 6}
len(Omega)   # 6 - מספר התוצאות האפשריות

# במאגר נתונים:
import pandas as pd
df = pd.read_csv('orders.csv')
len(df)    # מרחב המדגם של ההזמנות
🎯 אנלוגיה 2
כל הטיולים האפשריים מעיר
מתל-אביב אפשר לנסוע לכל יעד בארץ. מרחב המדגם של הנסיעות הוא כל היעדים האפשריים. אם תרצה לחשב 'מה ההסתברות שמישהו ייסע לאילת?', תצטרך קודם להגדיר את כל היעדים האפשריים - את Ω.
📐 מיפוי לנוסחה
  • כל יעד אפשרי = תוצאה ב-Ω
  • מרחב המדגם = רשימת היעדים
  • מרחב מדגם סופי = רשימה סגורה
  • מרחב מדגם אינסופי = רצף ערכים אפשריים

שאלה: מטילים שתי קוביות הוגנות בעלות 6 פאות. מהו גודל מרחב המדגם Ω, וכמה תוצאות מקיימות את התנאי שסכום הפאות שווה ל-7?
פתרון (תקציר): 1) מרחב המדגם הוא כל הזוגות הסדורים (a, b) כאשר a, b ∈ {1, 2, 3, 4, 5, 6}
📂 פרוס חישוב מלא
1) מרחב המדגם הוא כל הזוגות הסדורים (a, b) כאשר a, b ∈ {1, 2, 3, 4, 5, 6} 2) מספר התוצאות במרחב המדגם: |Ω| = 6 × 6 = 36 3) מאתרים את הזוגות שסכומם 7: (1,6), (2,5), (3,4), (4,3), (5,2), (6,1) 4) סופרים את הזוגות המתאימים: 6 תוצאות 5) התוצאה הסופית: |Ω| = 36, ומספר הזוגות שסכומם 7 הוא 6

אירוע

Event
A ⊆ Ω
💡 בקצרה
תת-קבוצה של מרחב המדגם - אוסף תוצאות שמעניינות אותנו. למשל, 'הטלת קובייה והמספר זוגי' הוא אירוע A = {2, 4, 6} ⊆ {1,2,3,4,5,6}.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
קטגוריה במלאי
חנות עם 100 מוצרים. רוצים לבדוק כמה מהם נמכרו במחיר מעל 100 ש"ח. הקטגוריה הזו היא 'אירוע' - תת-קבוצה של כל המוצרים. אם 25 מוצרים עלו יותר מ-100 ש"ח, אז |A|=25.
📐 מיפוי לנוסחה
  • כל המוצרים = Ω
  • מוצרים יקרים = A ⊆ Ω
  • |A| = מספר המוצרים באירוע
  • P(A) = |A|/|Ω| = 25/100 = 0.25
💻 כך זה נראה בקוד
import pandas as pd
products = pd.DataFrame({'price': [50, 120, 80, 200, 30]})

# אירוע A: מוצר במחיר > 100
A = products[products['price'] > 100]
P_A = len(A) / len(products)
# 2/5 = 0.4
🎯 אנלוגיה 2
תוצאה בכרטיס הגרלה
הגרלה עם 1000 מספרים. 'אירוע הזכייה' הוא אוסף המספרים הזוכים - אולי 10 מספרים. ההסתברות לזכייה היא 10/1000 = 1%. כל קבוצת מספרים שמעניינת אותנו (זוגיים, שלמים בריבוע) היא אירוע נפרד.
📐 מיפוי לנוסחה
  • 1000 מספרים = Ω
  • מספרי זכייה = אירוע A
  • |A| = 10, |Ω| = 1000
  • P(זכייה) = |A|/|Ω| = 0.01
💻 כך זה נראה בקוד

לסיווג, כל קטגוריה היא אירוע. במערכות זיהוי הונאה: 'עסקה חשודה' = אירוע. ההסתברות שעסקה היא הונאה = P(הונאה). המודל לומד לחזות את הסיכוי שכל דגימה שייכת לאירוע.

שאלה: מטילים קובייה הוגנת בעלת 6 פאות. הגדר את האירוע A = "התקבל מספר זוגי". מהי הסתברות האירוע A?
פתרון (תקציר): 1) מרחב המדגם: Ω = {1, 2, 3, 4, 5, 6}, כלומר |Ω| = 6\n
📂 פרוס חישוב מלא
1) מרחב המדגם: Ω = {1, 2, 3, 4, 5, 6}, כלומר |Ω| = 6\n 2) מזהים את האיברים השייכים לאירוע A (מספרים זוגיים): A = {2, 4, 6}\n 3) סופרים את גודל האירוע: |A| = 3\n 4) מחשבים הסתברות לפי הנוסחה הקלאסית: P(A) = |A| / |Ω|\n 5) מציבים: P(A) = 3 / 6 = 0.5\n 6) התוצאה: P(A) = 1/2

שלושת סוגי ההסתברות

Classical, Frequentist, Bayesian
קלאסית: P(A) = |A| / |Ω|
תדירותית: P(A) = lim_{n→∞} (כמה פעמים A קרה) / n
בייסיאנית: P(A) = מידת אמונה סובייקטיבית
💡 בקצרה
שלוש פילוסופיות שונות להגדרת מהי הסתברות. הקלאסית מתאימה למקרים סימטריים (קובייה, מטבע). התדירותית מתאימה לתופעות חוזרות. הבייסיאנית מאפשרת לכלול ידע מוקדם.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
שלוש דרכים לחזות תוצאת מבחן
תלמיד מתקרב למבחן. (1) קלאסית: מתוך 5 ציונים אפשריים, סיכוי שווה לכל אחד = 20%. (2) תדירותית: היסטורית, 60% מהתלמידים עוברים → P(מעבר)=60%. (3) בייסיאנית: 'אני סטודנט טוב, השקעתי, אז אני חושב 80%'.
📐 מיפוי לנוסחה
  • קלאסית = הנחת סימטריה (כל תוצאה שווה)
  • תדירותית = ספירה מהיסטוריה
  • בייסיאנית = שילוב היסטוריה + אמונה אישית
  • כל גישה מתאימה לסיטואציה אחרת
💻 כך זה נראה בקוד
# קלאסית: הטלת קובייה הוגנת

# תדירותית: סימולציה

tosses = np.random.randint(1, 7, 10000)
P_six_frequentist = (tosses == 6).mean()
# ≈ 0.166 (קרוב ל-1/6)
🎯 אנלוגיה 2
שלוש דרכים להעריך עסקה
סוחר רוצה להעריך הסתברות שעסקה תסתיים בהצלחה. (1) קלאסית: 'אם יש 3 תוצאות אפשריות שוות, P=1/3'. (2) תדירותית: 'מתוך 100 עסקאות דומות בעבר, 70 הצליחו → P=0.7'. (3) בייסיאנית: 'הניסיון הקודם אומר 70%, אבל החברה הזו אמינה, אז P=85%'.
📐 מיפוי לנוסחה
  • מקור הידע משתנה בין הגישות
  • קלאסית = ידע מתמטי מוחלט
  • תדירותית = רק נתוני העבר
  • בייסיאנית = ידע + נתונים + עדכון
💻 כך זה נראה בקוד
# גישה בייסיאנית: שילוב prior עם data
prior_belief = 0.85   # האמונה הראשונית
evidence = 0.70       # מה הדאטה אומר
# Bayesian updating: ממוצע משוקלל
weight_prior = 0.4
posterior = (weight_prior * prior_belief +
             (1-weight_prior) * evidence)
# = 0.76

שאלה: בקובייה הוגנת בעלת 6 פאות, חשב את ההסתברות הקלאסית לקבל מספר זוגי. בנוסף, בניסוי שבו הוטלה הקובייה 200 פעמים והתקבל מספר זוגי ב-110 מהפעמים, חשב את ההסתברות האמפירית. מהו ההפרש בין השתיים?
פתרון (תקציר): 1) הסתברות קלאסית (classical): מספר התוצאות המתאימות הוא 3 (2, 4, 6) מתוך 6 תוצאות אפשריות
📂 פרוס חישוב מלא
1) הסתברות קלאסית (classical): מספר התוצאות המתאימות הוא 3 (2, 4, 6) מתוך 6 תוצאות אפשריות 2) P_קלאסית = 3/6 = 0.5 3) הסתברות אמפירית (empirical): מחושבת לפי תדירות יחסית בניסוי 4) P_אמפירית = 110/200 = 0.55 5) ההפרש: 0.55 - 0.5 = 0.05 6) הערה: הסוג השלישי הוא הסתברות סובייקטיבית (subjective) — מבוססת על אמונה אישית ולא על חישוב

אקסיומות קולמוגורוב

Kolmogorov Axioms
1. P(A) ≥ 0 (אי-שליליות)
2. P(Ω) = 1 (נורמליזציה)
3. אם A ∩ B = ∅: P(A ∪ B) = P(A) + P(B)
💡 בקצרה
שלושת הכללים שמגדירים מהי הסתברות. כל פונקציה שמכבדת את שלושת האקסיומות הללו היא הסתברות תקפה. הם הבסיס המתמטי לכל תורת ההסתברות.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
חוקי חלוקת תקציב
חברה מחלקת תקציב של 100% למחלקות. (1) כל מחלקה מקבלת ≥ 0 (לא שלילי). (2) הסכום הכולל = 100% (כל התקציב מחולק). (3) שתי מחלקות שלא חופפות = הסכום שלהן הוא סכום נפרד.
📐 מיפוי לנוסחה
  • התקציב הכולל = 1 = P(Ω)
  • מחלקה i מקבלת P(Aᵢ) ≥ 0
  • סך החלוקה = 100% (אקסיומה 2)
  • מחלקות זרות מתחברות פשוט (אקסיומה 3)
💻 כך זה נראה בקוד
# אקסיומות בקוד
P = {'גשם': 0.3, 'שמש': 0.5, 'עננים': 0.2}

# 1. אי-שליליות:
assert all(p >= 0 for p in P.values())

# 2. סכום = 1:
assert sum(P.values()) == 1

# 3. איחוד זרים:
P_not_sunny = P['גשם'] + P['עננים']   # 0.5
🎯 אנלוגיה 2
חוקי משחק הוגן
משחק קוביות הוגן: (1) ההסתברות לכל מספר חייבת להיות אי-שלילית - אי אפשר 'להסתברות שלילית'. (2) ההסתברות שיצא משהו (1-6) = 1. (3) ההסתברות ליצא '1 או 2' = P(1) + P(2) (כי הם זרים).
📐 מיפוי לנוסחה
  • אקסיומה 1: בלי הסתברויות שליליות
  • אקסיומה 2: סך כל ההסתברויות = 1
  • אקסיומה 3: אירועים זרים מתחברים פשוט
  • תקפים לכל תופעה הסתברותית
💻 כך זה נראה בקוד
# בדיקת תקפות הסתברות
def is_valid_probability(P_dict):
    non_neg = all(p >= 0 for p in P_dict.values())
    sum_one = abs(sum(P_dict.values())-1) < 1e-9
    return non_neg and sum_one

P_valid = {'A': 0.3, 'B': 0.5, 'C': 0.2}
is_valid_probability(P_valid)   # True

כל פונקציית הסתברות במודל ML (softmax, sigmoid, התפלגות אחורית) חייבת לקיים את האקסיומות. אם המודל מחזיר ערכים שמפרים אותן - הוא לא 'תקין' הסתברותית.
✏️ תרגיל ופתרון
שאלה: במרחב הסתברות עם המאורעות A, B, C זרים בזוגות (disjoint) נתון: P(A) = 0.25, P(B) = 0.40, P(C) = 0.15. בנוסף ידוע ש-A ∪ B ∪ C = Ω. בדוק האם הנתונים עקביים עם אקסיומות קולמוגורוב, ואם לא — חשב את הערך החסר כדי שיהיו עקביים.
פתרון (תקציר): 1) אקסיומה 1 (אי-שליליות): כל הערכים בין 0 ל-1, מתקיים.
📂 פרוס חישוב מלא
1) אקסיומה 1 (אי-שליליות): כל הערכים בין 0 ל-1, מתקיים. 2) אקסיומה 2 (נרמול): P(Ω) חייב להיות שווה ל-1. 3) אקסיומה 3 (אדיטיביות עבור מאורעות זרים): P(A ∪ B ∪ C) = P(A) + P(B) + P(C) 4) מציבים: P(A ∪ B ∪ C) = 0.25 + 0.40 + 0.15 5) מסכמים: 0.25 + 0.40 = 0.65 6) ממשיכים: 0.65 + 0.15 = 0.80 7) מקבלים P(Ω) = 0.80, אך נדרש P(Ω) = 1 — לכן הנתונים לא עקביים. 8) ההפרש החסר: 1 - 0.80 = 0.20 9) מסקנה: כדי לקיים את אקסיומות קולמוגורוב צריך להוסיף הסתברות של 0.20 לאחד המאורעות (למשל P(C) = 0.35).

הסתברות מותנית

Conditional Probability
P(A|B) = P(A ∩ B) / P(B)
💡 בקצרה
ההסתברות של A בהינתן ש-B קרה. 'לצמצם את העולם רק ל-B', ובתוכו לבדוק מה אחוז A. דורש P(B) > 0.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
סינון לחיפוש אישי
באתר היכרויות, רוצים לדעת 'מה ההסתברות שמועמד הוא רופא, בהינתן שהוא בן 30+'. צובעים רק את האנשים בני 30+, ובתוכם בודקים כמה רופאים. אם 1000 משתמשים, 200 בני 30+, ו-50 מהם רופאים → P(רופא | 30+) = 50/200 = 25%.
📐 מיפוי לנוסחה
  • Ω = כל המשתמשים (1000)
  • B = 30+ (200 אנשים)
  • A ∩ B = רופאים בני 30+ (50)
  • P(A|B) = 50/200 = 0.25
  • אנחנו 'מצמצמים' את העולם ל-B
💻 כך זה נראה בקוד

מבחן רפואי - דיוק מותנה
מחלה נדירה (1% באוכלוסייה). בדיקה מזהה חולים ב-99% מהמקרים. שאלה: אם הבדיקה חיובית, מה הסיכוי שאני באמת חולה? התשובה לא 99% - היא הרבה יותר נמוכה. זה כי אנחנו צריכים P(חולה|חיובי), ולא P(חיובי|חולה).
📐 מיפוי לנוסחה
  • P(חיובי|חולה) = 0.99 (רגישות הבדיקה)
  • P(חולה) = 0.01 (שכיחות במחלה)
  • P(חולה|חיובי) ≠ P(חיובי|חולה)!
  • צריך משפט בייס כדי להפוך את הכיוון
💻 כך זה נראה בקוד

                       P_disease / P_pos)
# ≈ 0.167 (רק 16.7%!)
🤖 ב-ML
במערכות סיווג, המודל לרוב מחזיר P(class|features) - הסתברות מותנית של המחלקה בהינתן הפיצ'רים. זו המהות של רגרסיה לוגיסטית, Naive Bayes, ומודלי probabilistic classification.

שאלה: בכיתה יש 30 תלמידים. 18 מהם לומדים מתמטיקה, 12 לומדים פיזיקה, ו-8 לומדים את שני המקצועות. תלמיד נבחר באקראי ונמצא שהוא לומד מתמטיקה. מה ההסתברות שהוא לומד גם פיזיקה?

משפט בייס

Bayes' Theorem
P(A|B) = P(B|A) · P(A) / P(B)
💡 בקצרה
מאפשר 'להפוך' הסתברות מותנית: לחשב P(A|B) מתוך P(B|A) ועוד ידע על P(A) ו-P(B). הבסיס לעדכון אמונות לאור ראיות חדשות.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
עדכון אמונה לפי ראיות
רופא חושב מראש שיש סיכוי 5% שלמטופל יש מחלה X (Prior). הוא מבצע בדיקה והתוצאה חיובית (Evidence). עכשיו עליו לעדכן את האמונה שלו (Posterior). אם הבדיקה אמינה ב-95%, האמונה הסופית עולה משמעותית, אבל לא ל-95% - אלא תלויה ב-Prior.
📐 מיפוי לנוסחה
  • P(A) = Prior (האמונה הראשונית, 5%)
  • P(B|A) = Likelihood (אם A נכון, מה הסיכוי לראות B)
  • P(B) = Evidence (כמה סביר B באופן כללי)
  • P(A|B) = Posterior (אמונה מעודכנת)
  • Bayes מחבר את ארבעת הרכיבים
💻 כך זה נראה בקוד
# חישוב Posterior
prior = 0.05            # P(disease)
likelihood = 0.95       # P(positive|disease)
false_positive = 0.10   # P(positive|healthy)

evidence = (likelihood * prior +
            false_positive * (1-prior))
posterior = likelihood * prior / evidence
# ≈ 0.333 (33.3%, עלה מ-5%)
🎯 אנלוגיה 2
מסנן ספאם במייל

📐 מיפוי לנוסחה
  • P(spam) = 0.30 (Prior - שכיחות הספאם)
  • P(word|spam) = 0.80 (Likelihood)
  • P(word|not spam) = 0.05
  • P(word) = חישוב לפי שתי האפשרויות
  • P(spam|word) = Posterior
💻 כך זה נראה בקוד

P_word_given_not_spam = 0.05

Naive Bayes Classifier מבוסס ישירות על המשפט. גם רגרסיה לוגיסטית מחשבת Posterior. Bayesian Deep Learning משלב Prior על משקלי הרשת. בכל מערכת ML שעובדת תחת אי-ודאות, בייס הוא הכלי הבסיסי.
✏️ תרגיל ופתרון
שאלה: בדיקה רפואית מזהה מחלה בהסתברות 0.95 כאשר החולה באמת חולה, ונותנת תוצאה חיובית בהסתברות 0.10 כאשר החולה בריא. שכיחות המחלה באוכלוסייה היא 0.02. מהי ההסתברות שאדם שקיבל תוצאה חיובית באמת חולה?
פתרון (תקציר): 1) מגדירים: P(חולה) = 0.02, P(חיובי | חולה) = 0.95, P(חיובי | בריא) = 0.10
📂 פרוס חישוב מלא
1) מגדירים: P(חולה) = 0.02, P(חיובי | חולה) = 0.95, P(חיובי | בריא) = 0.10 2) מחשבים P(בריא) = 1 - 0.02 = 0.98 3) מחשבים את המונה: P(חיובי | חולה) · P(חולה) = 0.95 · 0.02 = 0.019 4) מחשבים את החלק השני של המכנה: P(חיובי | בריא) · P(בריא) = 0.10 · 0.98 = 0.098 5) סוכמים למכנה (הסתברות כוללת לתוצאה חיובית): 0.019 + 0.098 = 0.117 6) מציבים בנוסחת בייס: P(חולה | חיובי) = 0.019 / 0.117 7) התוצאה: P(חולה | חיובי) ≈ 0.1624 (כ-16.24%)

עצמאות סטטיסטית

Statistical Independence
A ⊥ B ⟺ P(A ∩ B) = P(A) · P(B)
💡 בקצרה
שני אירועים בלתי-תלויים אם ידיעת אחד לא משנה את ההסתברות של השני. שווה ערך: P(A|B) = P(A). אבחנה חשובה: עצמאות ≠ חוסר קורלציה (במקרים מסוימים).
📊 המחשה ויזואלית
🎯 אנלוגיה 1
שתי הטלות מטבע נפרדות
מטילים מטבע פעמיים. תוצאת ההטלה הראשונה לא משפיעה על השנייה - הן 'בלתי תלויות'. ההסתברות לקבל 'עץ ועץ' היא P(עץ) · P(עץ) = 0.5 · 0.5 = 0.25.
📐 מיפוי לנוסחה
  • אירוע A = הטלה ראשונה עץ
  • אירוע B = הטלה שנייה עץ
  • P(A) = P(B) = 0.5
  • P(A ∩ B) = P(A) · P(B) = 0.25
  • מטבע 'אינו זוכר' את ההטלות הקודמות
💻 כך זה נראה בקוד
import numpy as np

P_HH = ((toss1=='H') & (toss2=='H')).mean()
P_H = (toss1=='H').mean()
# P_HH ≈ P_H * P_H ≈ 0.25

תלות בין מזג אוויר לתנועה
האם 'גשם' ו'פקקים' עצמאיים? לא! P(פקקים|גשם) > P(פקקים) - גשם מגביר פקקים. אז הם תלויים. לעומת זאת, 'יום החודש' ו'מזג אוויר' עצמאיים: אין סיבה שיום 15 בחודש יהיה גשום יותר מאחר.
📐 מיפוי לנוסחה
  • A = יום החודש, B = גשם
  • אם A ⊥ B: P(B|A) = P(B) - אין השפעה
  • אם תלויים: P(B|A) ≠ P(B)
  • בדיקה מספרית: השוואת P(A∩B) ל-P(A)·P(B)

✏️ תרגיל ופתרון
שאלה: נתון P(A) = 0.4, P(B) = 0.5 ו-P(A ∩ B) = 0.2. האם המאורעות A ו-B בלתי תלויים סטטיסטית?
פתרון (תקציר): 1) תנאי העצמאות: A ו-B בלתי תלויים אם ורק אם P(A ∩ B) = P(A) · P(B)
📂 פרוס חישוב מלא
1) תנאי העצמאות: A ו-B בלתי תלויים אם ורק אם P(A ∩ B) = P(A) · P(B) 2) מחשבים את המכפלה: P(A) · P(B) = 0.4 · 0.5 = 0.2 3) משווים לנתון: P(A ∩ B) = 0.2 4) קיבלנו 0.2 = 0.2, כלומר התנאי מתקיים 5) המסקנה: המאורעות A ו-B בלתי תלויים סטטיסטית

פרמוטציות

Permutations
P(n, k) = n! / (n-k)!
💡 בקצרה
מספר הסידורים השונים של k פריטים מתוך n, כשהסדר חשוב. ל-3 ספרים על מדף יש 3! = 6 סידורים שונים. אם בוחרים 2 מתוך 3 לסידור: P(3,2) = 3·2 = 6.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
סידור ספרים על מדף
5 ספרים שונים, רוצים לסדר במדף עם 3 מקומות. למקום הראשון: 5 אפשרויות. אחרי בחירה - למקום השני: 4. אחרי שתי בחירות - לשלישי: 3. סה"כ: 5·4·3 = 60 סידורים אפשריים.
📐 מיפוי לנוסחה
  • n = 5 (כמות הספרים האפשרית)
  • k = 3 (מקומות במדף)
  • P(5,3) = 5!/(5-3)! = 5!/2! = 5·4·3 = 60
  • הסדר חשוב: ספר A במקום 1 ≠ ספר A במקום 2

סידור ראש-תור באירוע

📐 מיפוי לנוסחה
  • 10 שחקנים = n
  • 3 מיקומים מדורגים = k
  • מיקום ראשון: 10 אפשרויות
  • מיקום שני: 9 אפשרויות
  • מיקום שלישי: 8 אפשרויות

שאלה: בכמה דרכים שונות אפשר לסדר בשורה את 5 האותיות A, B, C, D, E?
פתרון (תקציר): 1) מספר הסידורים של n עצמים שונים בשורה הוא n! (פרמוטציה)
📂 פרוס חישוב מלא
1) מספר הסידורים של n עצמים שונים בשורה הוא n! (פרמוטציה) 2) כאן n = 5, לכן צריך לחשב 5! 3) מציבים: 5! = 5 × 4 × 3 × 2 × 1 4) מחשבים שלב אחר שלב: 5 × 4 = 20 5) ממשיכים: 20 × 3 = 60 6) ממשיכים: 60 × 2 = 120 7) ממשיכים: 120 × 1 = 120 8) התוצאה: קיימות 120 דרכים שונות לסידור

צירופים

Combinations
C(n, k) = n! / (k! · (n-k)!)
💡 בקצרה
מספר הדרכים לבחור k פריטים מתוך n, כשהסדר אינו חשוב. לבחור 3 ספרים מתוך 5 (בלי קשר לסדר) = C(5,3) = 10. C(n,k) קטן מ-P(n,k) כי לא סופרים את הסידור בתוך הקבוצה.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
בחירת חברים לועדה
מועדון עם 10 חברים, צריך לבחור ועדה של 3. הסדר לא משנה - ועדה {אבי, בני, גלי} זהה ל-{גלי, אבי, בני}. כמה ועדות שונות אפשר? C(10,3) = 120. הרבה פחות מ-P(10,3)=720, כי כל ועדה נספרה 6 פעמים בפרמוטציות.
📐 מיפוי לנוסחה
  • 10 חברים = n
  • 3 בועדה = k
  • פרמוטציות: 10·9·8 = 720
  • אבל כל ועדה נספרה 3! = 6 פעמים
  • צירופים: 720 / 6 = 120
💻 כך זה נראה בקוד

PMF, PDF, CDF

PMF, PDF, CDF
PMF: P(X = k) (דיסקרטי)
PDF: f(x), P(a ≤ X ≤ b) = ∫_a^b f(x)dx (רציף)
CDF: F(x) = P(X ≤ x) (לכל סוג)
💡 בקצרה
שלוש פונקציות מרכזיות לתיאור התפלגות. PMF מחזירה הסתברות של ערך בודד (דיסקרטי). PDF מחזירה צפיפות (רציף - אינטגרל נותן הסתברות). CDF מחזירה הסתברות מצטברת.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
סטטיסטיקה של פגישות
ניתוח של 1000 פגישות במשרד. PMF: 'איזה אחוז מהפגישות נמשכו בדיוק 30 דקות'. PDF (אם זמן רציף): 'צפיפות הפגישות בערכים קרובים ל-30 דקות'. CDF: 'איזה אחוז מהפגישות נמשכו עד 30 דקות' (מצטבר).
📐 מיפוי לנוסחה
  • PMF (דיסקרטי): P(X = 30) - בדיוק 30
  • PDF (רציף): f(30) - צפיפות סביב 30
  • CDF: F(30) = P(X ≤ 30) - הכל עד 30
  • F תמיד עולה (אחד מ-0 ל-1)

  • p = P(עץ) - הסיכוי להצלחה
  • התוצאה היחידה האפשרית: 0 או 1
  • תוחלת = p, שונות = p(1-p)

import scipy.stats as stats

samples.var()    # ≈ 0.25 = p(1-p)

ממוצע גיאומטרי

Geometric Mean
GM = (x₁ · x₂ · … · xₙ)^(1/n)
💡 בקצרה
השורש ה-n של מכפלת n הערכים. מתאים לחישוב ממוצע של אחוזי גידול, יחסים, תשואות. תכונה חשובה: GM ≤ AM (ממוצע אריתמטי) תמיד, ושוויון רק כשכל הערכים זהים.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
תשואת השקעה
השקעה הניבה תשואה: שנה 1: +50%, שנה 2: -50%. הממוצע האריתמטי = 0% (טעות!). הממוצע הגיאומטרי = √(1.5 · 0.5)-1 = -13.4%. אם תשקיע 100 ₪, אחרי 2 שנים יהיו לך 100 · 1.5 · 0.5 = 75 ₪ - הפסד אמיתי של 25%.
📐 מיפוי לנוסחה
  • ערכים = פקטורי גידול (1.5, 0.5)
  • תשואה ממוצעת = √(1.5·0.5)-1 = -13.4%
  • AM היה נותן 0% (תשובה שגויה)
  • GM נכון לסדרת תשואות מצטברות
  • GM תמיד ≤ AM
💻 כך זה נראה בקוד

צמיחת אוכלוסייה
עיר גדלה: שנה 1: 10%, שנה 2: 20%, שנה 3: 5%. מה הצמיחה הממוצעת השנתית? AM = 11.67%. GM = (1.1·1.2·1.05)^(1/3)-1 = 11.5%. GM נכון יותר כי הוא משקף שגידול הוא תהליך מצטבר (פקטורי), לא חיבורי.
📐 מיפוי לנוסחה

  • צמיחה ממוצעת = 11.5% לשנה
  • אם נשתמש ב-AM נקבל קצת יותר ולא מדויק
  • GM משקף הצטברות נכונה

IQR ו-Outliers

IQR & Outlier Detection
IQR = Q3 - Q1
Outlier: x < Q1 - 1.5·IQR או x > Q3 + 1.5·IQR
💡 בקצרה
IQR (Interquartile Range): הטווח בין הרבעון העליון לתחתון - הפיזור של 50% האמצעיים. עמיד בפני outliers. מבחן ה-1.5·IQR הוא דרך סטנדרטית לזיהוי חריגים בדאטה.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
טווח 'נורמלי' בכיתה
כיתת תלמידים. Q1=70 (תחתון), Q3=90 (עליון). IQR = 20. 'טווח נורמלי' = 70-90. תלמיד עם 50 (Q1-1.5·20 = 40, אז 50 לא קיצוני) או 95 (90 + 1.5·20 = 120, אז 95 נורמלי). אבל תלמיד עם 30 או 130 = outlier.
📐 מיפוי לנוסחה
  • Q1 = 25% מהנתונים מתחת לכך
  • Q3 = 75% מהנתונים מתחת לכך
  • IQR = Q3-Q1 = הפיזור של 50% האמצעיים
  • סף תחתון = Q1-1.5·IQR
  • סף עליון = Q3 + 1.5·IQR
  • מעל/מתחת לסף = outlier
💻 כך זה נראה בקוד
import numpy as np
scores = np.array([30, 70, 75, 78, 82, 85, 90, 130])
Q1, Q3 = np.percentile(scores, [25, 75])
IQR = Q3-Q1
lower = Q1-1.5 * IQR
upper = Q3 + 1.5 * IQR
outliers = scores[(scores < lower) | (scores > upper)]
# outliers = [30, 130]

אסימטריה

Skewness
💡 בקצרה
מדד לאי-סימטריה של ההתפלגות. Right skew (חיובי): זנב ארוך מימין, רוב הנתונים בערכים נמוכים. Left skew (שלילי): זנב משמאל, רוב בערכים גבוהים. סקיו 0 = סימטרי (נורמלי, לדוגמה).
📊 המחשה ויזואלית
🎯 אנלוגיה 1
הכנסות במדינה
התפלגות הכנסות לרוב Right-Skewed: רוב התושבים סביב הממוצע נמוך, אבל יש זנב ארוך של מעט עשירים מאוד. בגרף: ההר מתחיל בצד שמאל, יורד לאט לימין. בכזה התפלגות, הממוצע > החציון תמיד.
📐 מיפוי לנוסחה
  • Right skew (חיובי): mean > median
  • Left skew (שלילי): mean < median
  • סימטרי: mean = median
  • הזנב הארוך = הצד עם הסקיו
  • Right skew = זנב לימין
💻 כך זה נראה בקוד
import numpy as np
from scipy.stats import skew

# הכנסות (Right-skewed)
incomes = np.random.lognormal(8, 1, 10000)
skew(incomes)    # > 0 (Right skew)
np.mean(incomes), np.median(incomes)
# mean > median
🎯 אנלוגיה 2
ציוני מבחן קל
מבחן קל: רוב התלמידים מצליחים (ציונים 80-100), מעט נכשלים. גרף: זנב ארוך לשמאל (ציונים נמוכים), רוב בצד הימני. Left skew. במבחן קשה ההפך - Right skew, רוב נכשלים.
📐 מיפוי לנוסחה
  • מבחן קל = Left skew (זנב שמאלה)
  • מבחן קשה = Right skew (זנב ימינה)
  • מבחן מאוזן = סימטרי (skew ≈ 0)
  • כיוון הזנב מספר על אופי הבעיה
  • מודלים שמניחים נורמליות נכשלים בסקיו
💻 כך זה נראה בקוד

skew(easy_test)    # < 0 (Left skew)

# ציוני מבחן קשה

skew(hard_test)    # > 0 (Right skew)
🤖 ב-ML
פיצ'רים עם סקיו גבוה לרוב דורשים טרנספורמציה (log, sqrt, Box-Cox) לפני מודלינג. רגרסיה לינארית מניחה שגיאות נורמליות (סקיו 0). ב-financial data, סקיו חיובי מתואר ע"י log-normal distribution.
✏️ תרגיל ופתרון
שאלה: נתון מדגם של 5 ערכים: [2, 3, 4, 5, 11]. חשב את מקדם האסימטריה (Skewness) לפי הנוסחה: g₁ = (1/n) · Σ((xᵢ - x̄)/s)³, כאשר s הוא סטיית התקן עם חלוקה ב-n.
פתרון (תקציר): 1) מחשבים את הממוצע: x̄ = (2 + 3 + 4 + 5 + 11) / 5 = 25 / 5 = 5
📂 פרוס חישוב מלא
1) מחשבים את הממוצע: x̄ = (2 + 3 + 4 + 5 + 11) / 5 = 25 / 5 = 5 2) מחשבים סטיות מהממוצע: -3, -2, -1, 0, 6 3) מעלים בריבוע: 9, 4, 1, 0, 36 4) סוכמים: 9 + 4 + 1 + 0 + 36 = 50 5) שונות עם חלוקה ב-n: 50 / 5 = 10 6) סטיית תקן: s = √10 ≈ 3.162 7) מחשבים סטיות בחזקת שלוש: (-3)³ = -27, (-2)³ = -8, (-1)³ = -1, 0³ = 0, 6³ = 216 8) סוכמים: -27 + (-8) + (-1) + 0 + 216 = 180 9) מחלקים ב-s³: s³ = (√10)³ ≈ 31.623, ולכן 180 / 31.623 ≈ 5.692 10) מחלקים ב-n: g₁ = 5.692 / 5 ≈ 1.138 11) התוצאה: g₁ ≈ 1.14, כלומר אסימטריה חיובית (זנב ימני) בגלל הערך 11 שמושך את ההתפלגות ימינה

קורטוזיס

Kurtosis
💡 בקצרה
מדד ל'משקל הזנבות' של ההתפלגות. Heavy tails (kurtosis > 0): הסתברות גבוהה לאירועים קיצוניים. Light tails (kurtosis < 0): התפלגות 'שטוחה' באמצע ועם מעט קיצוניות. נורמלית: kurtosis = 0 (excess).
📊 המחשה ויזואלית
🎯 אנלוגיה 1
סיכון בשוק ההון
התפלגות תשואות יומיות במניה: אם kurtosis גבוה - אומר שיש 'הרבה ימים בלי כלום, ולעיתים זעזועים גדולים'. זנבות כבדים. אם kurtosis נמוך - תנודתיות יחסית קבועה. משבר 2008 הוא דוגמה לזנב כבד שלא נחזה ע"י מודלים שהניחו נורמליות.
📐 מיפוי לנוסחה
  • kurtosis גבוה = זנבות כבדים, סיכון אירוע נדיר וחמור
  • kurtosis נמוך = יציבות יחסית בקיצונים
  • נורמלית: kurtosis = 3 (regular), 0 (excess)
  • ב-fat tails: התפלגויות t, Cauchy, Lévy
  • סיכון: אם מניחים נורמליות אבל הזנב כבד - מטעה
💻 כך זה נראה בקוד
from scipy.stats import kurtosis

# תשואות עם זנבות כבדים
returns_heavy = np.random.standard_t(df=3, size=10000)
kurtosis(returns_heavy)   # > 0 (heavy tails)

# נורמלית
returns_normal = np.random.normal(0, 1, 10000)
kurtosis(returns_normal)  # ≈ 0
🎯 אנלוגיה 2
תדירות תאונות
באזור מסוים, רוב הימים אין תאונות. אבל מדי פעם יש 'יום שחור' עם הרבה תאונות. ההתפלגות של מספר תאונות יומי עם kurtosis גבוה - רוב הזמן קרוב לאפס, אבל זנב ארוך של ימים קיצוניים.
📐 מיפוי לנוסחה
  • אירועים נדירים אבל חמורים = kurtosis גבוה
  • ההפך מהתפלגות נורמלית 'אדישה'
  • ב-anomaly detection: kurtosis הוא סימן לתופעה
  • מודלים עם נורמליות לא יזהו את הזנבות הללו
  • צריך התפלגות כמו Student-t במקום
💻 כך זה נראה בקוד

ב-Financial ML: קורטוזיס גבוה דורש מודלים עם זנבות כבדים. ב-anomaly detection: התפלגויות עם kurtosis גבוה צריכות thresholds מתאימים. רגרסיה רגילה נכשלת בנוכחות זנבות כבדים - צריך Robust Regression.
✏️ תרגיל ופתרון
שאלה: נתון מדגם: [2, 4, 4, 4, 6]. הממוצע הוא 4 וסטיית התקן (אוכלוסייתית) היא σ = √(8/5). חשב את הקורטוזיס לפי הנוסחה: K = (1/n)·Σ(xᵢ - μ)⁴ / σ⁴.
פתרון (תקציר): 1) מחשבים סטיות מהממוצע: 2-4 = -2, 4-4 = 0, 4-4 = 0, 4-4 = 0, 6-4 = 2
📂 פרוס חישוב מלא
1) מחשבים סטיות מהממוצע: 2-4 = -2, 4-4 = 0, 4-4 = 0, 4-4 = 0, 6-4 = 2 2) מעלים בחזקת 4: (-2)⁴ = 16, 0⁴ = 0, 0⁴ = 0, 0⁴ = 0, 2⁴ = 16 3) סוכמים: 16 + 0 + 0 + 0 + 16 = 32 4) מחלקים ב-n = 5: 32/5 = 6.4 (זה הרגע הרביעי μ₄) 5) מחשבים σ⁴: σ² = 8/5 = 1.6, ולכן σ⁴ = 1.6² = 2.56 6) מחלקים: K = 6.4 / 2.56 = 2.5 7) התוצאה: הקורטוזיס הוא K = 2.5 (נמוך מ-3, כלומר ההתפלגות פלטיקורטית - שטוחה יחסית לנורמלית)

Z-score

Z-score (Standard Score)
z = (x - μ) / σ
💡 בקצרה
כמה סטיות תקן x רחוק מהממוצע. Z-score של 2 = שני σ מעל הממוצע. שימושי לסטנדרטיזציה (להוריד יחידות), לזיהוי outliers (|z| > 3), ולחישוב הסתברויות (דרך CDF).
📊 המחשה ויזואלית
🎯 אנלוגיה 1
השוואה בין מבחנים שונים
תלמיד קיבל 80 במתמטיקה (μ=70, σ=10) ו-75 בפיזיקה (μ=60, σ=15). באיזה הוא יחסית טוב יותר? Z-מתמטיקה = (80-70)/10 = 1. Z-פיזיקה = (75-60)/15 = 1. שווה ערך! Z מאפשר השוואה הוגנת.
📐 מיפוי לנוסחה
  • x - μ = הסטייה מהממוצע
  • (x - μ) / σ = הסטייה בסטיות תקן
  • Z = 0: בדיוק על הממוצע
  • Z = 1: σ אחד מעל
  • Z = -1: σ אחד מתחת
💻 כך זה נראה בקוד
math_score = (80-70) / 10    # 1.0
physics_score = (75-60) / 15 # 1.0
# שניהם 1σ מעל הממוצע - שווה ערך
🎯 אנלוגיה 2
זיהוי outliers בסנסור
מערכת ניטור חום בשרת. ממוצע 65°C, σ = 5°C. אם פתאום מתקבל קריאה 90°C, אז Z = (90-65)/5 = 5. כל |Z| > 3 מסומן כ-anomaly. ז"א, רק 0.3% מהזמן צפויה תוצאה כזו במצב נורמלי - נשמע התרעה.
📐 מיפוי לנוסחה
  • מדד התרעה: |Z| > 3 → outlier
  • Z = 5 = אירוע מאוד נדיר (< 0.0001%)
  • Z-score בודק חריגות סטנדרטיות
  • מבוסס על הנחת נורמליות
  • לזנבות כבדים: צריך שיטה אחרת
💻 כך זה נראה בקוד
# Anomaly detection בזמן אמת
import numpy as np
historical = np.random.normal(65, 5, 10000)
mu = historical.mean()
sigma = historical.std()

def check_anomaly(reading):
    z = (reading-mu) / sigma
    return abs(z) > 3, z

check_anomaly(90)   # (True, 5.0)
🤖 ב-ML
Standardization בפיצ'רים: x_new = (x - μ) / σ. כל הפיצ'רים אחרי StandardScaler הם Z-scores. גם משמש ב-outlier detection בסיסי, ובחישוב p-values במבחני השערות.
✏️ תרגיל ופתרון
שאלה: בכיתה ממוצע הציונים הוא μ = 75 וסטיית התקן היא σ = 8. תלמיד קיבל ציון x = 91. חשב את ה-Z-score שלו.
פתרון (תקציר): 1) נוסחת Z-score: z = (x - μ) / σ

Law of Large Numbers (LLN)
x̄ₙ → μ כש-n → ∞
💡 בקצרה
LLN (Law of Large Numbers, בעברית: חוק המספרים הגדולים) — ככל שגודל המדגם גדל, ממוצע המדגם מתכנס לתוחלת האמיתית. הסיבה למה ב-1000 הטלות מטבע נקבל יחס קרוב מאוד ל-50-50, גם אם ב-10 הטלות זה יכול להיות 7-3.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
הטלת מטבע רבות
10 הטלות: יכול להיות 7 ראשי-עץ, 3 פלי (70-30). 100 הטלות: צפוי 50-60 ראשי-עץ (קרוב יותר ל-50%). 10,000 הטלות: 4950-5050 (כמעט בדיוק 50%). ככל ש-n גדל, ההסתברות הניסיונית מתקרבת לתיאורטית.
📐 מיפוי לנוסחה

  • n גדול = שונות נמוכה
  • x̄ₙ = ממוצע מ-n דגימות
  • מתכנס ל-μ (התוחלת התיאורטית)
  • מספק לחישוב הסתברויות אמפיריות

Law of Small Numbers (Poisson 1837)
Binomial(n, p) ⟶ Poisson(λ), כאשר n → ∞, p → 0, n·p = λ קבוע
P(X = k) = (λᵏ · e⁻λ) / k!
💡 בקצרה
חוק המספרים הקטנים הוא משפט גבול הקובע שהתפלגות בינומית עם n גדול מאוד ו-p קטן מאוד, כך שהמכפלה λ = n·p נשארת קבועה ומתונה, מתכנסת להתפלגות פואסון. בפועל זה אומר שספירת אירועים נדירים על פני מספר גדול של נסיונות מתפלגת פואסון, גם אם ה-n המקורי עצום. בניגוד לחוק המספרים הגדולים שעוסק בהתכנסות הממוצע, החוק הזה עוסק בהתכנסות ההתפלגות עצמה.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
תאונות דרכים בצומת ספציפית
בצומת מסוימת עוברים כ-50,000 רכבים בשבוע, אבל הסיכוי שרכב יהיה מעורב בתאונה הוא כ-0.00004. n גדול, p זעיר, ו-λ = n·p = 2 תאונות בממוצע לשבוע. ספירת התאונות בכל שבוע לא תהיה בינומית מסורבלת אלא תתפלג Poisson(2). זה החוק בפעולה: אירוע נדיר, הרבה הזדמנויות, ספירה פואסונית.
📐 מיפוי לנוסחה
  • n = 50,000 רכבים בשבוע
  • p = 0.00004 (סיכוי תאונה לרכב)
  • λ = n·p = 2 תאונות בממוצע
  • P(X=k) = הסתברות ל-k תאונות השבוע
  • P(X=0) = e⁻² ≈ 0.135 (כ-13.5% מהשבועות ללא תאונות)

משפט הגבול המרכזי

Central Limit Theorem (CLT)
x̄ₙ ~ N(μ, σ²/n) עבור n גדול מספיק
💡 בקצרה
CLT (Central Limit Theorem, בעברית: משפט הגבול המרכזי) — אחד המשפטים המכריעים בסטטיסטיקה. ממוצעי דגימות מתפלגים נורמלית, ללא קשר להתפלגות המקורית של הדאטה. הסיבה למה נורמלית כל-כך נפוצה.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
ערפול של פעולות אקראיות
אם תיקח 30 מספרים אקראיים מהתפלגות מוזרה כלשהי, ותחשב את ממוצעם, ותחזור על זה הרבה פעמים - ההיסטוגרמה של הממוצעים תיראה נורמלית. ככל שגודל הדגימה (n) גדל, הקירוב לנורמלית טוב יותר.
📐 מיפוי לנוסחה
  • התפלגות מקורית = יכולה להיות כל דבר
  • n דגימות = גודל המדגם
  • ממוצע של דגימה = משתנה אקראי
  • התפלגות הממוצעים = נורמלית!
  • תוחלת = μ, סטיית תקן = σ/√n

הבסיס לכל מבחני השערה (t-tests, z-tests). למה ניתן להשתמש בנורמלית כתחליף לכל התפלגות עבור ממוצעי דגימה. גם הסיבה ש-random initialization במודלים עובד - הסכום של הרבה משקלים קטנים מתפלג נורמלית.
✏️ תרגיל ופתרון
שאלה: משתנה מקרי X בעל תוחלת μ = 50 ושונות σ² = 36. דוגמים מדגם בגודל n = 100. חשב את ההסתברות המקורבת ש-X̄ > 51 לפי משפט הגבול המרכזי. (נתון: Φ(1.67) ≈ 0.9525)
פתרון (תקציר): 1) לפי CLT, X̄ ~ N(μ, σ²/n) בקירוב\n
📂 פרוס חישוב מלא
1) לפי CLT, X̄ ~ N(μ, σ²/n) בקירוב\n 2) מחשבים את התוחלת של X̄: E[X̄] = 50\n 3) מחשבים את השונות של X̄: Var(X̄) = 36/100 = 0.36\n 4) מחשבים את סטיית התקן: σ_X̄ = √0.36 = 0.6\n 5) מתקננים לציון Z: Z = (51 - 5 0) / 0.6 = 1/0.6 ≈ 1.67\n 6) מחשבים את ההסתברות: P(X̄ > 5 1) = P(Z > 1.6 7) = 1 - Φ(1.67)\n 7) מציבים: 1 - 0.9525 = 0.0475\n 8) התוצאה: P(X̄ > 5 1) ≈ 0.0475

Standard Error

Standard Error
SE = σ / √n
💡 בקצרה
סטיית התקן של ממוצע הדגימה. מודד עד כמה הממוצע של המדגם מדויק כאומדן למוצע האוכלוסייה. ככל ש-n גדל, SE קטן - הדגימה מדויקת יותר.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
דיוק במצלמה
מצלמה צריכה לכוון לאוכלוסייה (מטרה). מצלמה עם n קטן (25 פיקסלים) - תמונה מטושטשת (SE גבוה). מצלמה עם n גדול (1000 פיקסלים) - תמונה חדה (SE קטן). הדיוק עולה כשורש n.
📐 מיפוי לנוסחה
  • σ = סטיית התקן של האוכלוסייה
  • n = גודל הדגימה
  • SE = σ / √n = דיוק הממוצע
  • n × 4 ⇒ SE / 2 (לא חצי - 4x!)
  • הגדלת דיוק עולה ב-n × 4
💻 כך זה נראה בקוד
import numpy as np
sigma = 10   # סטיית תקן באוכלוסייה
for n in [25, 100, 400, 1600]:
    SE = sigma / np.sqrt(n)
    print(f'n={n}: SE={SE:.2f}')
# n=25: SE=2.00
# n=100: SE=1.00  (חצי)
# n=400: SE=0.50  (רבע)
# n=1600: SE=0.25 (1/8)
🎯 אנלוגיה 2
החלטה על גודל מדגם בסקר
מכון רוצה לעשות סקר עם דיוק של 1%. אם σ של ההתפלגות הוא 10%, אז SE רצוי = 1%. לפי הנוסחה: n = (σ/SE)² = 100. אם דיוק 0.5% נדרש - n = 400. כל הכפלה של דיוק עולה פי 4 בגודל המדגם.
📐 מיפוי לנוסחה
  • דיוק רצוי = SE
  • σ = ידוע מ-pilot studies
  • n = (σ/SE)² (מהפיכת הנוסחה)
  • הכפלת דיוק = פי 4 בגודל הדגימה
  • Trade-off: דיוק נוסף עולה זמן וכסף

Population vs Sample

💡 בקצרה

📊 המחשה ויזואלית
🎯 אנלוגיה 1

רווח סמך

Confidence Interval (CI)
CI = x̄ ± Z · (σ/√n) או x̄ ± t · (s/√n)
💡 בקצרה
טווח של ערכים שלפיו מעריכים את הפרמטר באוכלוסייה. CI של 95% פירושו: אם נחזור על הדגימה 100 פעמים, 95 מהפעמים הטווח שלנו יכיל את הפרמטר האמיתי. פירוש מטעה: 'יש 95% סיכוי שהממוצע בטווח'.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
טווח אומדן בדייג
דייג רוצה להעריך כמה דגים יש באגם. הוא יודע שעם 95% וודאות, מספר הדגים בין 8,000 ל-12,000. CI = [8000, 12000]. הוא לא יודע את המספר המדויק, אבל יודע את הטווח שמכיל אותו ברוב המקרים.
📐 מיפוי לנוסחה
  • המספר האמיתי = μ (לא ידוע)
  • אומדן = x̄ = למשל 10,000
  • Margin of error = Z × SE
  • CI = x̄ ± margin
  • רמת הסמך 95% = הסיכוי שהשיטה תכיל את μ

📐 מיפוי לנוסחה
  • 45% = x̄ (האומדן)

  • 42% - 48% = CI
  • חפיפת CI = אין הבדל מובהק
  • CI לא חופפים = הבדל מובהק

התפלגות t של סטודנט

Student's t-Distribution
💡 בקצרה
התפלגות דמוית-נורמלית, אבל עם זנבות כבדים יותר. משמשת כשאינך יודע את σ של האוכלוסייה וצריך להעריך אותו ממדגם קטן (n < 30). פרמטר יחיד: degrees of freedom = n - 1. כש-df → ∞, מתקרבת לנורמלית.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
אי-ודאות נוספת באומדן
כשאתה יודע את σ - אתה משתמש בנורמלית. כשאתה לא יודע ומשתמש ב-s כאומדן - יש 'אי-ודאות נוספת' שמוסיפה לזנבות. ככל ש-n גדל, האומדן s יותר מדויק, וההתפלגות מתקרבת לנורמלית.
📐 מיפוי לנוסחה
  • df = n-1 (degrees of freedom)
  • df קטן → זנבות כבדים יותר → t רחב יותר
  • df גדול → t מתקרב לנורמלית
  • df = ∞: t = N(0,1) בדיוק
  • מתאים למדגמים קטנים
💻 כך זה נראה בקוד
import scipy.stats as stats
import numpy as np

# השוואת ערכי t וערכי z
for df in [5, 30, 100, 1000]:
    t_critical = stats.t.ppf(0.975, df)
    z_critical = stats.norm.ppf(0.975)
    print(f'df={df}: t={t_critical:.3f}, z={z_critical:.3f}')
# df=5: t=2.571 (רחב יותר)
# df=1000: t≈1.962 (קרוב ל-z=1.96)
🎯 אנלוגיה 2
מבחן עם מעט תלמידים
מבחן שנערך על 5 תלמידים בלבד - אי-אפשר להסיק הרבה. סטיית התקן של המדגם s לא מדויקת ל-σ של האוכלוסייה. ה-t-distribution מתאים את עצמו לכך, ומוסיף 'גמישות' (זנבות כבדים) להתחשב באי-הוודאות.
📐 מיפוי לנוסחה
  • n קטן ⇒ s לא מדויק
  • אי-וודאות נוספת ⇒ זנבות כבדים
  • df = n-1 (5-1=4 לדוגמה)
  • t-test מטפל באי-וודאות הזו
  • מדגם גדול: אין הבדל מנורמלית

השערת אפס והשערה חלופית

Null & Alternative Hypothesis (H₀, H₁)
H₀: מצב ברירת מחדל · H₁: השערת המחקר
💡 בקצרה
H₀ (השערת האפס) היא הטענה המוקדמת שאנו מנסים לדחות - בדרך כלל 'אין הבדל' או 'אין אפקט'. H₁ (השערה חלופית) היא הטענה החדשה שהחוקר מנסה להוכיח. רק אם הראיות חזקות מספיק - דוחים את H₀.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
חזקת החפות במשפט
במשפט פלילי: הנאשם 'חף מפשע עד שיוכח אחרת'. זה H₀ - חזקת החפות. התביעה צריכה להציג ראיות חזקות מספיק כדי לדחות אותה (H₁ = 'הוא אשם'). אם הראיות חלשות - לא מרשיעים, אבל זה לא אומר שהוא חף מפשע - רק שלא הוכח.
📐 מיפוי לנוסחה
  • H₀ = חזקת החפות (בנקודת המוצא)
  • H₁ = אשם (מה שצריך להוכיח)
  • ראיות = הדאטה שאספנו
  • סף ההרשעה = α (רמת המובהקות)

💡 בקצרה
מדד לעוצמת ההבדל בין שתי קבוצות, מנורמל לסטיית התקן. d ≈ 0.2 = קטן, 0.5 = בינוני, 0.8 = גדול. בניגוד ל-p-value, גודל האפקט לא תלוי בגודל המדגם.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
ההבדל בין שני בתי ספר
בית ספר א' עם ממוצע 80, בית ספר ב' עם ממוצע 82. הבדל של 2 נקודות. האם זה הרבה או מעט? תלוי בשונות! אם σ=1: d=2 (אפקט ענק). אם σ=20: d=0.1 (אפקט קטן). Cohen's D מנרמל את ההבדל לסטיית התקן, מאפשר השוואה עקבית.
📐 מיפוי לנוסחה
  • x̄₁-x̄₂ = ההבדל המוחלט
  • s_pooled = סטיית תקן משוקללת
  • d מציב את ההבדל בקנה מידה
  • d=0.2: 85% חפיפה בין הקבוצות
  • d=0.8: 53% חפיפה (הבדל ברור)

import numpy as np

def cohens_d(x, y):
    nx, ny = len(x), len(y)
    sx, sy = np.std(x, ddof=1), np.std(y, ddof=1)
    s_pooled = np.sqrt(((nx-1)*sx**2 + (ny-1)*sy**2) / (nx+ny-2))
    return (np.mean(x)-np.mean(y)) / s_pooled

group_a = np.array([80, 82, 78, 79, 81])
group_b = np.array([85, 87, 83, 84, 86])

שאלה: נתונות שתי דגימות בלתי תלויות בעלות שונויות שוות: דגימה 1 עם n₁ = 5, ממוצע x̄₁ = 12, שונות מדגמית s₁² = 4. דגימה 2 עם n₂ = 5, ממוצע x̄₂ = 10, שונות מדגמית s₂² = 6. חשב את ערך סטטיסטי המבחן t.

מבחן Welch

Welch's t-Test
💡 בקצרה
וריאנט של מבחן t לשתי דגימות שלא דורש שוויון שונויות בין הקבוצות. בעולם האמיתי, נדיר שהשונויות שוות בדיוק - לכן הרבה ממליצים להשתמש ב-Welch כברירת מחדל.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
השוואה בין קבוצות עם פיזורים שונים
השוואת זמני משלוח: מחברה A (פיזור נמוך, יציבה) לעומת B (פיזור גבוה, לפעמים מהירה לפעמים איטית). מבחן t רגיל מניח שהפיזורים שווים - לא נכון כאן. Welch מתאים: לוקח בחשבון את ההבדל ביציבות.
📐 מיפוי לנוסחה
  • Welch מאפשר σ₁ ≠ σ₂
  • מחשב SE בצורה אחרת מ-t רגיל
  • degrees of freedom משתנים גם
  • תוצאה: יותר חזק כשהשונויות שונות
  • מבחן t רגיל מטעה במצב כזה
💻 כך זה נראה בקוד

# רגיל
t_reg, p_reg = stats.ttest_ind(A, B, equal_var=True)
🎯 אנלוגיה 2
השוואת קבוצות בגדלים שונים
קבוצה ניסיון של 200 משתמשים מול קבוצת בקרה של 50 משתמשים. גדלים שונים → סבירות גבוהה לשונויות שונות. Welch מטפל גם בגדלים שונים וגם בשונויות שונות. רוב הא/ב טסטים בפועל זה Welch ולא רגיל.
📐 מיפוי לנוסחה
  • n₁ ≠ n₂ ⇒ סיכוי גבוה ל-σ₁ ≠ σ₂
  • Welch לא דורש n או σ זהים
  • ברירת מחדל ב-python/scipy
  • מאבד מעט power כשהשונויות שוות באמת
  • מומלץ במצב של אי-ודאות

Prior, Likelihood, Posterior

Prior, Likelihood, Posterior
Posterior ∝ Likelihood × Prior
💡 בקצרה
הליבה של הסקה בייסיאנית. Prior = אמונה לפני הדאטה. Likelihood = כמה הדאטה מסתדר עם השערה. Posterior = האמונה המעודכנת אחרי הדאטה. תהליך הלמידה הוא עדכון Prior ל-Posterior באמצעות Likelihood.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
רופא שמעדכן אבחנה
רופא רואה מטופל. לפני הבדיקה: 'יש 10% שיש לו שפעת' (Prior). מבצע בדיקה - חיובית. עכשיו מעדכן: 'יש 70% שיש לו שפעת' (Posterior). ה-Likelihood היא 'כמה סביר שהבדיקה הזו תצא חיובית אם זו שפעת'.
📐 מיפוי לנוסחה
  • Prior = P(שפעת) = 0.10 לפני הבדיקה
  • Likelihood = P(חיובי | שפעת) = 0.95
  • Evidence = P(חיובי) = הכל יחד
  • Posterior = P(שפעת | חיובי) = 0.70 אחרי
  • האמונה מתעדכנת לפי הראיות
💻 כך זה נראה בקוד

prior = 0.10
likelihood_pos_given_disease = 0.95
likelihood_pos_given_healthy = 0.10  # false positive

evidence = (likelihood_pos_given_disease * prior +
            likelihood_pos_given_healthy * (1-prior))
posterior = (likelihood_pos_given_disease * prior /
             evidence)
# posterior ≈ 0.514

Types of Priors
💡 בקצרה
Informative (אינפורמטיבי): כשיש ידע קודם עשיר. Weakly Informative: ידע חלש. Uninformative/Flat: אין ידע קודם, הכל שווה. Conjugate Prior: בחירה מתמטית שמייצרת Posterior באותה משפחה כמו Prior (חישוב אנליטי).

🎯 אנלוגיה 1
רופא ותיק לעומת מתחיל
רופא מומחה רואה מטופל - יש לו ידע מקדים עשיר (Informative Prior): 'בגיל הזה, עם הסימפטומים האלה, סביר 70% שזו מחלה X'. רופא מתחיל: 'אני לא יודע, ניתן הסתברות שווה לכל אבחנה' (Flat Prior). שניהם יגיעו לאבחנה דרך הראיות, אבל המומחה יגיע מהר יותר ועם פחות בדיקות.

  • Weakly Informative = ידע כללי בלבד
  • Flat = אין ידע, כל אפשרות שווה
  • Conjugate = בחירה נוחה חישובית
  • ידע קודם 'חוסך' בדיקות

Peeking Problem

Peeking Problem
💡 בקצרה
מלכודת נפוצה בא/ב טסטינג: 'הצצה' בנתונים תוך כדי הניסוי ועצירה מוקדם אם רואים תוצאה רצויה. זה מנפח את α (False Positive) - הסיכוי לתוצאה 'מובהקת' מקרית עולה דרמטית.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
הגרלה עם בדיקות חוזרות
מטילים מטבע ובודקים אם 'הוגן' עם α=5%. אם בודקים 100 פעמים תוך כדי, סביר שלפחות פעם אחת נראה הבדל 'מובהק' במקרה. עוצרים ב'תוצאה הטובה' ומדווחים = הטעיה. גם ניסוי אמיתי שתעצור בפעם הראשונה שרואית 'הצלחה' - תפרסם הרבה תוצאות שגויות.
📐 מיפוי לנוסחה
  • כל הצצה = בדיקה סטטיסטית נוספת
  • 10 הצצות במקום אחת = α אפקטיבי גבוה
  • α=5% → אחרי 10 הצצות ≈ 40%
  • הפתרון: לקבוע גודל מדגם מראש
  • או: שיטות sequential testing (לא לעצור באמצע)
💻 כך זה נראה בקוד
# סימולציה של peeking
import numpy as np
from scipy import stats

false_positives_with_peeking = 0
n_trials = 1000
for _ in range(n_trials):
    A = np.random.normal(0, 1, 1000)
    B = np.random.normal(0, 1, 1000)
    # 10 הצצות במהלך הניסוי
    for n in range(100, 1001, 100):
        _, p = stats.ttest_ind(A[:n], B[:n])
        if p < 0.05:
            false_positives_with_peeking += 1
            break
# שיעור FP ≫ 5%
🎯 אנלוגיה 2

במשחק קלפים יש כללים: השופט סופר את הניקוד רק בסוף. אם השופט מסתכל באמצע ועוצר 'מי שמוביל' - השחקן השני לא מקבל הזדמנות אמיתית. כך גם בא/ב: עצירה מוקדמת = הטיה לטובת הקבוצה שמוביל באקראי בנקודת ההצצה.
📐 מיפוי לנוסחה
  • המשחק = הניסוי
  • הצצה = בדיקת מובהקות באמצע
  • עצירה מוקדמת = החלטה לפני סיום הניסוי
  • האמת המתפרסמת = מוטה
  • השאלה הנכונה: מה הולך לקרות בטווח הארוך

Multiple Comparisons Problem
💡 בקצרה
כשמבצעים הרבה מבחנים סטטיסטיים בו-זמנית, הסיכוי שלפחות אחד יצא 'מובהק' במקרה עולה דרמטית. בדיקת 20 פיצ'רים עם α=5% → 64% סיכוי לפחות תוצאה 'מובהקת' שגויה.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
הימור עם הרבה כרטיסים
מהמרים על 100 משחקים. הסיכוי לזכות בכל אחד נמוך - אבל לזכות באחד מתוך 100 גבוה מאוד. אם תפרסם רק את הזכייה ותתעלם משאר - תיראה כמומחה. במחקר: אם בודקים 100 פיצ'רים, סביר לראות 5 'מובהקים' במקרה (α=5%).
📐 מיפוי לנוסחה
  • Family-wise error rate = הסיכוי ש'לפחות אחד שגוי'
  • n מבחנים עם α כל אחד
  • P(לפחות 1 FP) ≈ 1-(1-α)^n
  • 20 בדיקות עם α=0.05: P(FP) ≈ 0.64
  • פתרונות: Bonferroni, FDR
💻 כך זה נראה בקוד
import numpy as np
from scipy import stats

# 20 פיצ'רים, אף אחד לא קשור באמת
X = np.random.normal(0, 1, (1000, 20))
y = np.random.normal(0, 1, 1000)

p_values = []
for i in range(20):
    _, p = stats.pearsonr(X[:, i], y)
    p_values.append(p)

# כמה 'מובהקים' עם α=0.05?
significant = sum(p < 0.05 for p in p_values)
# צפוי כ-1 (5% מ-20), לעיתים יותר
🎯 אנלוגיה 2
סורק רפואי שמוצא כל דבר
סורק MRI בודק 100 פרמטרים בגוף. אם עם α=5% נראה 5 'חריגים' - האם הם אמיתיים? לרוב לא, רק רעש. רופא מנוסה יודע: לא להגיב לכל ממצא, אלא רק לחריגים מאוד. פתרון מתמטי: Bonferroni correction (חלוקת α במספר הבדיקות).
📐 מיפוי לנוסחה
  • 100 פרמטרים = 100 מבחנים
  • α המקורי = 0.05
  • Bonferroni: α_מתוקן = 0.05 / 100 = 0.0005
  • רק חריגים מאוד נחשבים מובהקים
  • False Discovery Rate (FDR) - גישה מתונה יותר

🤖 ב-ML
ב-causal inference: confounders מייצרים את הפרדוקס הזה. תמיד צריך לחפש משתנים מסבירים. ב-decision trees: התת-מבנה לפי משתנים יוצר split שמונע פרדוקס. בא/ב: צריך randomization כדי לוודא שאין confounders.
✏️ תרגיל ופתרון
שאלה: בית חולים A טיפל ב-100 חולים קלים (80 הבריאו) ו-100 חולים קשים (30 הבריאו). בית חולים B טיפל ב-200 חולים קלים (170 הבריאו) ו-50 חולים קשים (10 הבריאו). חשב את אחוז ההחלמה בכל קבוצה ובסך הכל, והראה את הפרדוקס.
פתרון (תקציר): 1) אחוז החלמה בחולים קלים בבית חולים A: 80/100 = 80%
📂 פרוס חישוב מלא
1) אחוז החלמה בחולים קלים בבית חולים A: 80/100 = 80% 2) אחוז החלמה בחולים קלים בבית חולים B: 170/200 = 85% 3) אחוז החלמה בחולים קשים בבית חולים A: 30/100 = 30% 4) אחוז החלמה בחולים קשים בבית חולים B: 10/50 = 20% 5) בכל תת-קבוצה בית חולים B עדיף בקלים (85% > 80%) ו-A עדיף בקשים (30% > 20%) 6) סך החלמות בבית חולים A: 80 + 30 = 110 מתוך 200, כלומר 55% 7) סך החלמות בבית חולים B: 170 + 10 = 180 מתוך 250, כלומר 72% 8) פרדוקס סימפסון: בסך הכל B נראה טוב יותר (72% > 55%), אך זה נובע מכך ש-B טיפל ביחס גבוה יותר של חולים קלים (200/250 = 80%) לעומת A (100/200 = 50%) 9) מסקנה: יש לפלח לפי משתנה מבלבל (confounder) — חומרת המחלה — לפני השוואת אחוזי ההחלמה

מקדם קורלציה

Correlation Coefficient (Pearson)
r = Σ(xᵢ-x̄)(yᵢ-ȳ) / √(Σ(xᵢ-x̄)²·Σ(yᵢ-ȳ)²)
💡 בקצרה
מודד עוצמת הקשר הלינארי בין שני משתנים. ערכים בטווח [-1, 1]: 1=קורלציה חיובית מושלמת, -1=שלילית מושלמת, 0=אין קשר לינארי. ערך נמוך לא בהכרח אומר אין קשר - יכול להיות קשר לא-לינארי.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
גובה ומשקל
באוכלוסייה: ככל שגובה עולה, משקל נוטה לעלות. r ≈ 0.7 (קורלציה חיובית חזקה). אבל לא מושלם (r ≠ 1) - יש אנשים גבוהים ורזים, נמוכים וכבדים. הקורלציה מודדת את החוזק של הקשר הכללי.
📐 מיפוי לנוסחה
  • r=1: כל עלייה ב-x → עלייה פרופורציונלית ב-y
  • r=0: אין קשר לינארי
  • r=-1: עלייה ב-x → ירידה ב-y
  • 0 < |r| < 0.3: קשר חלש
  • 0.3 < |r| < 0.7: בינוני
  • |r| > 0.7: חזק
💻 כך זה נראה בקוד

  • אולי איכות → גם מחיר וגם ציון

🤖 ב-ML

✏️ תרגיל ופתרון
שאלה: נתונים שני משתנים: X = [1, 2, 3, 4] ו-Y = [2, 4, 5, 7]. חשב את מקדם הקורלציה של פירסון (Pearson) בין X ל-Y.

📂 פרוס חישוב מלא

Confounders, Mediators, Colliders

Confounders, Mediators, Colliders
💡 בקצרה
שלוש דרכים שבהן משתנה Z משפיע על קשר בין X ל-Y. Confounder: Z גורם לשני (סיבה משותפת). Mediator: X→Z→Y (Z 'מתווך' את הסיבתיות). Collider: X→Z←Y (שני המשתנים גורמים ל-Z).
📊 המחשה ויזואלית
🎯 אנלוגיה 1
שלושה מבני קשר
Confounder (גלידה ↔ כרישים): טמפרטורה גורמת לשניהם. Mediator (תרגיל → סרוטונין → מצב רוח): סרוטונין מתווך בין תרגיל למצב רוח. Collider (כישרון + מזל → הצלחה הוליוודית): שני גורמים בלתי-תלויים מובילים להצלחה.
📐 מיפוי לנוסחה
  • Confounder Z: X ← Z → Y
  • Mediator M: X → M → Y
  • Collider C: X → C ← Y
  • כל אחד דורש טיפול שונה במודל
  • בלי הבחנה - מסקנות שגויות

✏️ תרגיל ופתרון
שאלה: במחקר על קשר בין שתיית קפה (X) לבין מחלות לב (Y), נמדדו ההתאמות הבאות: - Corr(X, Y) = 0.40 ללא התאמה למשתנים. - Corr(X, Y | עישון) = 0.05 לאחר התאמה למשתנה Z = עישון. - Corr(X, Y | לחץ דם) = 0.38 לאחר התאמה למשתנה M = לחץ דם (משתנה שמושפע משתיית קפה). - Corr(X, Y | מאושפז) = -0.30 לאחר התאמה למשתנה C = מאושפז בבית חולים (מושפע גם מ-X וגם מ-Y). עבור כל אחד מהמשתנים Z, M, C קבע אם הוא Confounder, Mediator או Collider, והסבר האם נכון להתאים אליו.
פתרון (תקציר): 1) משתנה Z (עישון): ההתאמה הורידה את המתאם מ-0.40 ל-0.05, כלומר עישון "הסביר" את רוב הקשר בין X ל-Y.
📂 פרוס חישוב מלא
1) משתנה Z (עישון): ההתאמה הורידה את המתאם מ-0.40 ל-0.05, כלומר עישון "הסביר" את רוב הקשר בין X ל-Y. 2) עישון משפיע גם על שתיית קפה וגם על מחלות לב (חץ ל-X וחץ ל-Y), לכן Z הוא Confounder. 3) מסקנה ל-Z: חובה להתאים (control) כדי להסיר הטיה — האפקט האמיתי של קפה קרוב ל-0. 4) משתנה M (לחץ דם): שתיית קפה מעלה לחץ דם, ולחץ דם מעלה סיכון למחלות לב, כלומר X → M → Y. 5) M נמצא על המסלול הסיבתי בין X ל-Y, לכן M הוא Mediator. 6) מסקנה ל-M: אסור להתאים אם רוצים את האפקט הכולל (total effect) — התאמה חוסמת חלק מהאפקט האמיתי של X. 7) משתנה C (אשפוז): גם קפה וגם מחלות לב מגדילים סיכוי לאשפוז, כלומר X → C ו-Y → C. 8) C הוא צומת משותף של שני חצים נכנסים, לכן C הוא Collider. 9) מסקנה ל-C: אסור להתאים — ההתניה עליו יצרה מתאם שלילי מלאכותי (-0.30) שלא קיים באוכלוסייה (Selection Bias / Berkson's paradox). 10) סיכום: מתאמים רק ב-Z (Confounder); לא מתאמים ב-M (Mediator) ולא ב-C (Collider).

ATE, ATT, ATC, CATE

Causal Effect Types
ATE = E[Y(1) - Y(0)] (Average Treatment Effect)
ATT = E[Y(1) - Y(0) | T=1] (Average Treatment effect on Treated)
ATC = E[Y(1) - Y(0) | T=0]
CATE = E[Y(1) - Y(0) | X=x] (Conditional)
💡 בקצרה
ארבעה סוגי אפקטים סיבתיים. ATE: ההשפעה הממוצעת על האוכלוסייה. ATT: על מי שקיבל טיפול בפועל. ATC: על מי שלא קיבל. CATE: על תת-קבוצה ספציפית.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
תוכנית הכשרה בארגון
ארגון מציע תוכנית הכשרה. ATE = ההשפעה הממוצעת של ההכשרה על כלל העובדים (אם כולם היו מקבלים). ATT = ההשפעה על מי שבחר להתאמן בפועל (אולי 800$ עליה בשכר). ATC = ההשפעה על מי שלא התאמן (אולי 200$ אם היו) - כי אנשים שלא בחרו פחות יפיקו תועלת. ATE = ממוצע ATT ו-ATC.

שאלה: באוכלוסייה של 10 אנשים, 4 קיבלו טיפול (T=1) ו-6 לא (T=0). אפקטי הטיפול האינדיבידואליים (Y₁ - Y₀) הם: למטופלים [8, 6, 4, 2], ולא-מטופלים [3, 1, -1, 5, 2, 0]. חשב ATE, ATT ו-ATC.

📂 פרוס חישוב מלא

ניסוי מבוקר אקראי

Randomized Controlled Trial (RCT)
💡 בקצרה
סטנדרט הזהב להוכחת סיבתיות. מקצים משתתפים אקראית לקבוצת טיפול וקבוצת בקרה. הרנדומיזציה מבטלת confounders ידועים ולא-ידועים, ומאפשרת לזהות את ההשפעה הסיבתית של הטיפול.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
ניסוי תרופתי קלאסי
חברת תרופות בודקת תרופה חדשה. 1000 חולים מחולקים אקראית: 500 מקבלים תרופה (טיפול), 500 פלצבו (בקרה). הרנדומיזציה מבטיחה שגיל, מין, חומרת המחלה יתפלגו דומה. אם רואים שיפור רק בקבוצת הטיפול - זו סיבתיות אמיתית.
📐 מיפוי לנוסחה
  • הקצאה אקראית = מבטלת אפליה והטיות
  • קבוצת טיפול = T=1
  • קבוצת בקרה = T=0
  • Y(1)-Y(0) = ההשפעה הסיבתית
  • הרנדומיזציה הופכת קורלציה לסיבתיות

התאמת ציוני נטייה

Propensity Score Matching
💡 בקצרה
שיטה לחיקוי RCT כשאי-אפשר לבצע ניסוי בפועל. מחשבים לכל יחידה 'ציון נטייה' = P(טיפול | פיצ'רים), ומתאימים יחידות מקבוצת הטיפול ליחידות דומות בקבוצת הבקרה. בכך מנטרלים confounders.
📊 המחשה ויזואלית
🎯 אנלוגיה 1
השוואה הוגנת בלי ניסוי
רוצים לדעת האם תוכנית רווחה עוזרת. אי-אפשר להגריל אנשים אקראית לעוני. במקום: מוצאים אנשים דומים (גיל, השכלה, מיקום) - חלקם קיבלו רווחה וחלקם לא. משווים את התוצאות שלהם. כל זוג 'דומה ככל האפשר' - כמעט כמו ניסוי אקראי.
📐 מיפוי לנוסחה
  • Propensity Score = P(treatment | X)
  • כל יחידה בקבוצת הטיפול → מוצאים זוגות דומים בבקרה
  • ההשוואה רק בין יחידות עם ציון נטייה דומה
  • מנטרל confounders ידועים
  • לא מנטרל confounders לא-נמדדים

✏️ תרגיל ופתרון
שאלה: מודל לוגיסטי מחשב הסתברות לקבלת טיפול לפי P(T=1|X) = σ(-1 + 0.5·X). נתונים 4 פרטים: A (טופל, X=2), B (טופל, X=0), C (לא טופל, X=2), D (לא טופל, X=4). חשב את ציון הנטייה לכל פרט והתאם כל מטופל למבוקרת הקרובה ביותר לפי הציון.
פתרון (תקציר): 1) ציון נטייה לפרט A: η = -1 + 0.5·2 = 0, לכן e(A) = σ(0) = 1/(1+e⁰) = 0.5
📂 פרוס חישוב מלא
1) ציון נטייה לפרט A: η = -1 + 0.5·2 = 0, לכן e(A) = σ(0) = 1/(1+e⁰) = 0.5 2) ציון נטייה לפרט B: η = -1 + 0.5·0 = -1, לכן e(B) = σ(-1) = 1/(1+e¹) ≈ 0.269 3) ציון נטייה לפרט C: η = -1 + 0.5·2 = 0, לכן e(C) = σ(0) = 0.5 4) ציון נטייה לפרט D: η = -1 + 0.5·4 = 1, לכן e(D) = σ(1) = 1/(1+e⁻¹) ≈ 0.731 5) התאמה ל-A (e=0.5): מרחקים מ-C (0.5) ו-D (0.731) הם 0 ו-0.231, הקרוב ביותר הוא C 6) התאמה ל-B (e≈0.269): מרחקים מ-C (0.5) ו-D (0.731) הם 0.231 ו-0.462, הקרוב ביותר הוא C 7) תוצאה: A↔C, B↔C (C משמש כביקורת לשני המטופלים)
